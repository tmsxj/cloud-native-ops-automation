kubeadm 默认证书有效期一年，过期前建议通过集群升级或手动续期。在多 master 集群中，我会逐个节点执行 kubectl cordon、drain，然后 kubeadm certs renew all，再重启 kubelet，最后 uncordon。这样可以保证业务连续性。对于生产环境，我会配合 cron job 或升级策略实现自动续期，避免证书过期导致集群不可用。同时，kubelet 自身支持证书轮换，不需要额外干预
cordon：将节点标记为“不可调度”，即不让新的 Pod 调度到这个节点上。

drain：驱逐节点上现有的 Pod（除 DaemonSet 外），让它们优雅地迁移到其他节点。

kubelet：运行在每个节点上的代理，负责管理 Pod 的生命周期。

uncordon：恢复节点为可调度状态，与 cordon 相反。

cron job：定时任务，可以定期执行某条命令或脚本。

证书轮换：kubelet 在 client 证书快过期时自动向 API Server 申请新证书，无需人工介入。
