```shell
 ==================== 镜像打包与传输全流程（详细注释版） ====================
 以下命令按顺序在对应机器上执行，实现从美国服务器打包镜像并最终传到 Harbor 虚拟机
 注意：所有命令中的文件名、别名等需根据实际情况调整

 -------------------- 1. 在美国服务器打包镜像（保留标签） --------------------
 作用：将美国服务器上所有非 <none> 标签的镜像打包成一个压缩文件，并保留原始仓库名和标签。
 命令解析：
   docker images --format "{{.Repository}}:{{.Tag}}"      # 列出所有镜像的"仓库:标签"
   grep -v "<none>"                                       # 过滤掉无标签的镜像（防止打包无用的中间层）
   $( ... )                                               # 命令替换，将镜像列表作为参数传给 docker save
   docker save ...                                         # 将指定镜像保存为 tar 流（保留所有元数据，包括标签）
   | gzip > all-images-$(date +%Y%m%d).tar.gz              # 压缩并命名为带日期的文件
 执行位置：在美国服务器（us）的终端中执行。
docker save $(docker images --format "{{.Repository}}:{{.Tag}}" | grep -v "<none>") | gzip > all-images-$(date +%Y%m%d).tar.gz

 可选：查看打包后文件大小
ls -lh all-images-*.tar.gz

 -------------------- 2. 查看打包文件是否包含标签（在美国服务器执行） --------------------
 作用：检查打包文件内的 manifest.json，确认镜像的仓库名和标签是否被保留。
 方法一：使用 jq 工具格式化 JSON 输出（需安装 jq：sudo apt install jq -y）
   tar -xOzf ... manifest.json     # 从 tar.gz 中提取 manifest.json 并输出到 stdout
   jq .                             # 格式化 JSON 并高亮显示
 方法二：使用 python -m json.tool（无需安装额外工具）
   tar -xOzf ... manifest.json | python -m json.tool
tar -xOzf all-images-*.tar.gz manifest.json | jq .   # 推荐用 jq，输出更清晰

 -------------------- 3. 在腾讯云服务器上启动 tmux 后台传输（从美国服务器拉取） --------------------
 tmux 是一个终端复用器，可以在后台保持会话运行，即使断开 SSH 连接也不会终止其中的进程。
 安装 tmux：sudo apt install tmux -y
 操作步骤：
   - 创建新会话并命名为 rsync_from_us
   - 在会话中执行 rsync 命令（会自动显示进度）
   - 按 Ctrl+B 再按 D 脱离会话（detach），进程继续在后台运行
   - 之后可随时用 `tmux attach -t rsync_from_us` 重新附着查看进度

 登录腾讯云服务器后执行：
tmux new -s rsync_from_us
 此时会进入 tmux 会话，在其中执行以下 rsync 命令：
 rsync 参数说明：
   -a  归档模式，保留文件属性（递归、权限、时间等）
   -v  显示详细信息
   -P  等价于 --partial --progress，支持断点续传并显示进度
   us  美国服务器的 SSH 别名（需已配置免密登录）
   :/root/all-images-*.tar.gz  源文件路径
   /root/                       目标目录（腾讯云服务器的 /root）
rsync -avP us:/root/all-images-*.tar.gz /root/

 按 Ctrl+B 然后按 D 脱离会话

 -------------------- 4. 从腾讯云服务器传输到 Harbor 虚拟机（在腾讯云服务器执行，可另开 tmux） --------------------
 同样使用 tmux 保持后台运行，避免终端中断。
 创建第二个 tmux 会话（例如命名为 rsync_to_hb）：
tmux new -s rsync_to_hb
 在会话中执行：
   rsync -avP /root/all-images-*.tar.gz hb:/data/harbor/
   hb 是 Harbor 虚拟机的 SSH 别名（需已配置免密登录）
   /data/harbor/ 是目标目录（确保该目录存在，否则先执行 ssh hb "mkdir -p /data/harbor"）
rsync -avP /root/all-images-*.tar.gz hb:/data/harbor/

 按 Ctrl+B D 脱离会话

 -------------------- 5. 传输完成后验证文件一致性（在腾讯云服务器执行） --------------------
 作用：通过对比 MD5 哈希值，确保文件在传输过程中未损坏。
 命令解析：
   ssh us "md5sum /root/all-images-*.tar.gz"    # 在美国服务器上计算源文件 MD5
   md5sum /root/all-images-*.tar.gz             # 在腾讯云服务器上计算目标文件 MD5
 两个输出应该完全相同。

 先获取美国服务器上的 MD5：
ssh us "md5sum /root/all-images-*.tar.gz"
 再获取腾讯云服务器上的 MD5：
md5sum /root/all-images-*.tar.gz
 手动对比两个哈希值，一致则文件完整。

 -------------------- 6. 在 Harbor 虚拟机上加载镜像（登录 Harbor 虚拟机执行） --------------------
 作用：将压缩包解压并导入到 Harbor 虚拟机的本地 Docker 引擎，使镜像可用。
 登录 Harbor 虚拟机（假设别名为 hb）：
ssh hb
 进入镜像存放目录（如果不在，则 cd /data/harbor）：
cd /data/harbor
 加载镜像：
   gunzip -c ...      # 解压到标准输出
   | docker load      # 从标准输入读取镜像 tar 流并加载
gunzip -c all-images-*.tar.gz | docker load
 查看镜像列表，确认所有镜像的仓库名和标签都已正确显示：
docker images

 -------------------- 可选：清理临时文件 --------------------
 在腾讯云服务器上删除临时包（释放空间）：
rm -f /root/all-images-*.tar.gz
 在 Harbor 虚拟机上删除包（如果不需要保留）：
rm -f /data/harbor/all-images-*.tar.gz

 ==================== 结束 ====================
补充说明：

如果镜像列表过长导致 docker save $(...) 报错 Argument list too long，可采用文件列表方式：

bash
docker images --format "{{.Repository}}:{{.Tag}}" | grep -v "<none>" > image-list.txt
docker save $(cat image-list.txt) | gzip > all-images-$(date +%Y%m%d).tar.gz

jq 是一个轻量级的 JSON 处理工具，安装命令：sudo apt install jq -y。如果不安装，可用 python -m json.tool 代替，但输出格式略逊。
tmux 的基本操作：
tmux new -s 会话名：创建新会话
tmux attach -t 会话名：重新附着到会话
tmux ls：列出所有会话
在会话内，按 Ctrl+B 然后按 D 脱离
按 Ctrl+B 然后按 ? 查看所有快捷键帮助
rsync 的 -P 参数非常重要，它同时支持进度显示和断点续传，确保传输中断后重新运行命令会从断点继续。
```
