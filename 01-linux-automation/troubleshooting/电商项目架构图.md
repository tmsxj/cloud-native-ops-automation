🗺️ 下一步实施计划：AI 智能运维助手
目标

在现有私有云基础上，部署 Ollama（轻量级大模型） + openEuler MCP（运维知识库与工具框架），实现一个能回答运维问题、分析日志、检索历史案例的智能助手。

前置条件

已有 Kubernetes 集群（5节点）、Harbor 私有仓库、EFK 日志系统、Prometheus 监控。

美国服务器（us）可访问外网，用于下载模型和镜像。

至少一个 master 节点有 2Gi 以上空闲内存（可将 Ollama 和 MCP 分别调度到不同 master 节点）。

📦 阶段一：部署 Ollama 推理服务
1. 在美国服务器上获取轻量模型
bash
# 拉取 tinyllama 模型（约 600MB）
docker run --rm -v $(pwd)/models:/root/.ollama/models ollama/ollama:0.1.30 ollama pull tinyllama
# 打包模型文件
tar -czf ollama-models-tinyllama.tar.gz -C models .
2. 传输模型包到 Harbor 虚拟机，解压到 NFS 共享目录（或 local-path PVC）

假设 NFS 共享路径 /data/nfs/ai（需提前搭建 NFS）：

bash
rsync -avP ollama-models-tinyllama.tar.gz root@192.168.1.61:/tmp/
ssh hb "mkdir -p /data/nfs/ai && cd /data/nfs/ai && tar -xzf /tmp/ollama-models-tinyllama.tar.gz"
3. 部署 Ollama 到 K8s

创建命名空间 ai

使用之前提供的 YAML（修改镜像地址为 Harbor 路径，挂载 PVC，添加 tolerations 调度到 master 节点）

暴露服务（NodePort 或 Ingress）

4. 测试
bash
curl -X POST http://<nodeIP>:<NodePort>/api/generate -d '{"model": "tinyllama", "prompt": "What is Kubernetes?"}'
📚 阶段二：部署 openEuler MCP（轻量级知识库）
1. 获取 MCP 镜像或源码

由于该项目可能未提供官方镜像，你需要在美国服务器上构建镜像。步骤如下：

bash
git clone https://gitee.com/openeuler/log_anomaly_detection.git
cd log_anomaly_detection
# 根据 README 构建镜像，推送到 Harbor

（若已有镜像则直接使用）

2. 创建 K8s 部署 YAML

设置环境变量 ES_HOST 指向 elasticsearch.logging.svc.cluster.local:9200

设置 OLLAMA_URL 为 Ollama 服务的集群内地址（如 http://ollama.ai:11434）

挂载 PVC 存放知识库文档（可复用 NFS 或使用 local-path）

添加 tolerations 调度到另一 master 节点（避免资源竞争）

3. 导入历史故障案例

将你之前的排错笔记（如 Kubernetes 部署、Harbor 配置、监控问题等）整理成 Markdown 或文本文件，放入 PVC 的指定目录（如 /knowledge）。MCP 会自动索引。

4. 测试 MCP 与 Ollama 集成

通过 MCP 的 API 发送问题，观察是否能检索知识库并返回答案。例如：

bash
curl -X POST http://<MCP-Service>/ask -d '{"question": "如何排查 Kubernetes Pod 无法启动的问题？"}'
🔗 阶段三：集成监控告警（可选进阶）

在 Prometheus Alertmanager 中配置 Webhook，当某些告警触发时，调用 MCP 的 API 分析日志，并将结果通过 Webhook 发送到钉钉/邮件。

示例：当 frontend Pod 重启次数超过阈值时，触发分析最近一小时的 frontend 日志，并生成摘要。

📌 资源估算与调度
组件	内存请求	建议调度节点	备注
Ollama (tinyllama)	1.5Gi	master1	使用 tolerations
MCP (轻量模式)	1.0Gi	master2	使用 tolerations
合计	2.5Gi	—	master1 剩余约 1.5Gi，master2 剩余约 2.2Gi，足够

若内存不足，可临时关闭 MCP 的聚类功能，只保留知识库检索。
